关键观察：本地 Ollama 用 batch 反而更慢（0.5x）
──────────────────────────────────────────────────────────────

原因：
  - batch 内部使用线程池并发发送 3 条请求给同一个 Ollama 实例
  - 但 Ollama 共享同一块 GPU/CPU，3 条并发请求互相抢占算力
  - 并发反而引入了调度开销，导致总时间 > 顺序调用

对比：
  顺序 invoke   22.4s（单条结束后才开始下一条，GPU 独占）
  batch 并发    44.0s（3 条同时抢 GPU，互相等待，还有线程调度开销）

结论：
  - 本地单实例模型（Ollama）：batch / ainvoke+gather 不带来加速，甚至更慢
  - 远程独立服务器 API（如 HuggingFace、Qwen API）：每条请求打到独立机器
    → 参考 LLM_aInvoke_ConcurrentDemo_output.txt：远程并发加速比 2.8x
  - batch 的优势场景：远程 API + 大量请求（批量数据处理、离线评估等）
